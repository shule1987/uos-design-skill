## 平台基线
- 默认目标是 Deepin / UOS 桌面上的 Qt + DTK 应用。
- 窗口、弹窗、模糊和拖动行为必须结合目标 Qt 版本与 Wayland / X11 会话判断。
- 先参考 `references/platform-compatibility.md`，再决定是否使用 frameless、`Popup.Window` 或实时模糊。

## 控件优先级

**优先使用 DTK 控件**
- 优先使用 DTK (Deepin Toolkit) 提供的原生控件
- DTK 控件已集成系统主题、无障碍支持和性能优化
- 仅在 DTK 无法满足需求时才自定义实现

- 设计或实现组件时，先检查 DTK 是否已有对应能力，例如 `Dtk.Button`、`Dtk.Menu`、`Dtk.Dialog`。
- 只有当原生控件缺少目标能力，或桌面窗口结构确实需要更高定制度时，才退回到自定义 fallback。

## 命名规范

### 组件命名
- 使用 PascalCase：`BaseButton`, `TextField`, `PopupMenu`
- 组件文件名与组件名一致：`BaseButton.qml`
- 内部组件使用 `component` 关键字定义

### 属性命名
- 使用 camelCase：`backgroundColor`, `isActive`, `maxWidth`
- 布尔属性使用 `is/has/can` 前缀：`isVisible`, `hasError`, `canEdit`
- 信号使用动词：`clicked`, `textChanged`, `itemSelected`

### ID 命名
- 使用 camelCase：`mainWindow`, `sidebarPanel`, `submitButton`
- ID 应简短且语义明确
- 避免使用通用名称如 `item`, `rect`

## 窗口规范

### 标题栏
**默认优先保持窗管装饰语义，再追求统一头部**
- 主应用窗口默认优先保留由窗管管理的边框、阴影、圆角、拖动、缩放和其他标准窗口行为
- 如果目标 Qt / DTK / 桌面环境已经验证支持统一头部视觉且不损失上述窗管能力，可以采用标题栏与工具栏一体化布局
- 如果用户明确要求“默认采用标题栏与工具栏一体化，出现问题再 fallback 到系统装饰”，则不能只交付 fallback 方案；必须同时给出已验证的一体化默认路径与系统装饰回退条件，或先明确说明平台阻塞点
- 如果无法同时满足统一头部视觉和窗管装饰语义，则保留系统窗口装饰，并把工具栏紧贴标题栏下方
- 当要求“圆角、描边、投影必须由窗管接管”时，不要再从应用侧用 `D.DWindow.windowRadius`、`borderWidth`、`borderColor`、`shadowRadius`、`shadowOffset`、`shadowColor` 去调出一个“看起来像系统”的边框与投影
- 如果统一头部视觉只能通过应用侧阴影、应用侧圆角或应用侧描边来维持外观，则该路径不满足窗管接管要求，应退回系统装饰
- 不要把 `Qt.FramelessWindowHint` 作为 UOS / Deepin 主窗口的默认实现
- 只有在用户明确接受 client-side decoration 取舍，或平台特定 DTK 路径已被验证时，才使用纯自绘无边框窗口
- 所有应用左上角都必须保留应用 logo，且默认只显示 logo，不显示应用名称文本
- 如果需求文档中的任何要求与本设计规范冲突，以本设计规范为第一优先级；需要偏离时必须先明确说明冲突点和取舍原因

### 窗口控制按钮
**右上角标准按钮组（从左到右）**

1. 菜单按钮（可选）- 三横线图标
2. 最小化按钮 - 横线图标
3. 最大化/还原按钮 - 方框图标
4. 关闭按钮 - X 图标，悬停背景色 `Theme.danger`

- 窗口控制按钮作为一组连续操作排列，按钮之间不额外留缝，保证命中区域连续且整齐。
- 如果包含菜单按钮，它应与最小化、最大化、关闭等按钮共享同一垂直对齐基线。

### 主菜单与关于
- 所有负责弹出应用主菜单的按钮，一律使用 DTK 标准菜单按钮，不使用自定义 `Button`、`AppButton` 或其他自绘菜单触发器代替
- 应用主菜单的显示位置必须遵循 DTK 菜单按钮的默认弹出规范与附着关系，不要手工用窗口坐标计算主菜单的 `x`、`y` 位置
- 只要应用提供主菜单，主菜单中就必须包含主题切换入口；不要把主题切换只藏在设置页而主菜单缺失
- 主题切换至少提供三种模式：`跟随系统`、`浅色`、`深色`，名称可以本地化，但语义必须完整
- `跟随系统` 必须持续跟随系统深浅主题变化，而不是只在应用启动时读取一次系统主题后就固定下来
- 所有应用内对话框一律使用 DTK 提供的对话框类型；项目层只提供标题、正文、语义状态和动作信号，不接管对话框外框、按钮区、阴影、圆角、间距和其他视觉样式
- 所有“关于”入口弹出的关于窗口或关于对话框，一律使用 DTK 标准 `AboutDialog`
- 不要用自定义 `AppDialog`、普通 `Popup` 或其他自绘内容替代标准关于对话框，除非目标环境经验证确实缺失 DTK `AboutDialog`

## 视觉效果

### 毛玻璃效果
**主导航左侧栏的毛玻璃是硬要求，其他场景仍可渐进增强**
- 对承担桌面主导航的持久左侧栏，必须使用已验证的窗管或 compositor blur 路径
- 对这类左侧栏，纯色或半透明背景只能作为兼容回退，不能替代目标交付
- 对抽屉、浮层和非主导航面板，可继续按渐进增强决定是否启用 `WindowBlur` 或 `MultiEffect`
- 模糊半径建议：12-64，按面板大小选择
- 适用于：侧边栏、抽屉、浮动面板
- 所有 blur 方案都必须有纯色回退

- 毛玻璃更适合放在侧边栏、抽屉和浮动面板一类独立容器中，而不是大面积铺满整个主内容区。
- 当 blur 不可用时，直接退回 `Theme.bgPanel`、`Theme.panelBg` 等纯色或半透明背景。

### 阴影规范
- 卡片阴影：`blur: 20, spread: 0, offset: (0, 4), opacity: 0.08`
- 对话框阴影：`blur: 40, spread: 0, offset: (0, 8), opacity: 0.12`
- 悬浮菜单阴影：`blur: 24, spread: 0, offset: (0, 6), opacity: 0.1`
- 顶层主窗口如果要求使用窗管装饰效果，则不要用应用内阴影去替代窗管阴影。
- 顶层主窗口如果要求由窗管接管圆角、描边和投影，则不要通过应用侧 `D.DWindow` 阴影和边框参数继续做强度、半径或颜色微调。

## 布局规范

### 侧边栏
- 宽度：展开态通常为 200-240px，并可按内容和窗口宽度调节；不要把 60px 图标栏当作默认或唯一折叠目标
- 位置：当侧边栏承担桌面应用的主导航时，一律在左侧，并与右侧主内容形成固定左右分栏
- 当侧边栏承担桌面应用的主导航时，blur 是硬要求，不是可选项；必须使用已验证的窗管或 compositor blur 路径
- 导航项的 hover / selected 背景与 selected 前景必须来自主题层的语义变量，例如 `Theme.navItemHoverBg`、`Theme.navItemSelectedBg`、`Theme.navItemSelectedFg`
- 不要在侧边栏组件或页面中临时用 `Theme.mix(...)`、`Theme.withAlpha(...)` 等表达式拼接导航状态色；如需新状态色，先回到主题层定义变量
- 仅在导航层级较多或空间紧张时建议启用折叠功能；不是所有侧边栏都必须强制提供折叠
- 折叠按钮位置：logo 右侧 20px 处
- 折叠按钮图标：展开状态用 `chevron-left`，折叠状态用 `chevron-right`
- 折叠动画时长：200ms
- 可调节侧边栏达到最小宽度 100px 时，或用户主动点击折叠按钮时，应触发隐藏侧边栏
- 展开与收起必须使用动画，避免突兀跳变

### 内容区域
- 最小边距：16px
- 最大内容宽度：1200px（居中显示）
- 卡片间距：12-16px
- 选项组、设置组和卡片内部只保留建立层级、对齐、阅读和点击热区所必需的留白；不要加入大块无意义空白
- 内容区中的可见内容应作为一个整体相对内容区居中；当内容宽度被限制或小于可用宽度时，左外边距和右外边距必须一致
- 不要把主内容列、卡片组或仪表盘组合钉在内容区某一侧，然后在另一侧留下明显更宽的空白
- 对于“左说明 + 右控件”的重复行，不论它出现在设置页、启动项列表还是选项卡片组里，都应拆成稳定的左右两槽结构，而不是把开关、复选框、下拉框或按钮直接接在可变宽度文本后面
- 同一组重复行中的右侧控件必须共享同一条右对齐列线；标题长短、描述有无、徽标存在与否、控件禁用态或当前值差异都不应改变右侧控件列的位置
- 右侧控件列与背景右边缘的间距固定遵循该容器的右 inset；不要让某些行的控件只是“看起来靠右”但实际没有对到统一终点
- 同一横向内容带中的卡片、面板或统计块需要遵循统一网格；如果左侧是大卡片、右侧是多张堆叠卡片，则右侧堆叠后的总高度必须与左侧大卡片对齐
- 对明显属于同一组合区块的卡片，外轮廓的上、下、左、右边界都应形成稳定对齐关系，避免出现顶部齐了但底部参差或右侧整体悬空
- 横向标题区、概览区和操作区要考虑视觉重心；不要把标题、说明和主按钮全部堆在一侧，同时让另一侧留下大块空白
- 如果一侧信息密度明显更高，应通过补充状态块、摘要块、辅助操作或重新分配列宽来恢复左右视觉均衡

### 响应式断点
- 小屏：< 768px
- 中屏：768px - 1200px
- 大屏：> 1200px

## 交互规范

### 动画时长
- 快速反馈：120ms（按钮按下、开关切换）
- 标准过渡：200ms（菜单展开、标签页切换）
- 慢速动画：350ms（对话框、抽屉）

### 悬停状态
- 按钮悬停：背景色变化 + 120ms 过渡
- 列表项悬停：背景色 `Theme.surfaceHover`
- 链接悬停：下划线 + 颜色加深

### 焦点状态
- 焦点边框：2px solid `Theme.accentForeground`
- 焦点边框圆角：继承元素圆角 + 2px
- 焦点动画：200ms

## 无障碍规范

### 颜色对比度
- 正文文本：至少 4.5:1（WCAG AA）
- 大号文本（18px+）：至少 3:1
- 交互元素：至少 3:1

### 键盘导航
- 所有交互元素必须支持 Tab 键导航
- 支持 Enter/Space 激活
- 支持 Esc 关闭对话框/菜单

### 屏幕阅读器
- 所有图标按钮必须有 `Accessible.name`
- 表单控件必须有可见标签，必要时补充 `Accessible.description`
- 自定义控件必须设置合适的 `Accessible.role` 并同步状态变化

- 自定义组件需要明确声明自己的无障碍语义，包括名称、角色和关键状态变化。
- 当组件本身只是视觉容器时，不要让它抢占真正交互控件的无障碍焦点。

## 性能规范

### 列表渲染
- 超过 50 项使用 `ListView` 而非 `Repeater`
- 启用 `cacheBuffer` 优化滚动
- 使用 `reuseItems: true` 复用项

### 图片加载
- 使用 `asynchronous: true` 异步加载
- 大图使用 `sourceSize` 限制尺寸
- 启用 `cache: true` 缓存

### 动画优化
- 避免在动画中修改 `width`/`height`
- 优先使用 `scale`/`opacity`/`x`/`y`
- 使用 `layer.enabled: true` 启用硬件加速

## 主题适配

### 深浅主题切换
- 深浅主题切换以 `Theme.mode` 和 `dark` 状态为入口，由主题变量统一驱动。
- 业务组件只消费语义变量，不直接在组件内部维护独立的浅深色分支。
- 只要应用有主菜单，就应在主菜单中暴露 `跟随系统 / 浅色 / 深色` 主题切换，设置页可以保留同等能力，但不能替代主菜单入口。
- `Theme.mode === "system"` 时，界面应响应运行中的系统主题变化，而不是只在启动时同步一次。

### 主题色
- 系统活动色源：`Theme.systemAccent`
- 普通文字 / 图标：`Theme.textPrimary` / `Theme.iconNormal`
- 强调文字 / 悬停图标：`Theme.textStrong` / `Theme.iconStrong`
- 激活态前景：`Theme.accentForeground`
- 激活态背景：`Theme.accentBackground`
- 兼容别名：`Theme.accent` 等价于 `Theme.accentBackground`
- 成功色：`Theme.success`（绿色）
- 警告色：`Theme.warning`（橙色）
- 危险色：`Theme.danger`（红色）

### 自定义主题
- 支持接入系统活动色或用户自定义活动色
- 自动从系统活动色生成前景 / 背景两组变体
- 保持对比度符合无障碍标准

### 使用约定
- 标题、关键数值、悬停强调文案优先使用 `Theme.textStrong`
- 正文、菜单项、列表说明优先使用 `Theme.textPrimary`
- 次要说明、快捷键、提示信息使用 `Theme.textSecondary` 或 `Theme.textMuted`
- 默认交互图标使用 `Theme.iconNormal`；悬停或强调图标使用 `Theme.iconStrong`；选中导航、当前步骤和激活图标使用 `Theme.accentForeground`；禁用图标使用 `Theme.textDisabled`
- `Theme.textMuted` 只用于次要说明文本或极弱化信息，不应用作默认导航、工具栏、按钮、菜单或列表操作图标色
- 选中图标、链接、当前步骤、可点击激活文本使用 `Theme.accentForeground`
- 主按钮、选中项背景、分页当前页、进度填充使用 `Theme.accentBackground`
- 业务组件禁止直接写死蓝色字面量；任何蓝色都应通过 `Theme.systemAccent` 或其派生变量使用
- 生产 QML 中，中心主题文件以外禁止直接写十六进制颜色字面量；需要新颜色语义时，先在主题层增加变量，再由页面和组件消费

## 文本规范

### 字体与字号
- 默认采用系统 UI 字体，不在普通业务组件中写死字体族
- 字号不作为固定 px 规范写死，应跟随系统字号、系统缩放和无障碍设置变化
- 代码、日志、快捷键等确实需要对齐的场景可以单独指定等宽字体
- 标题、正文、说明文本的层级优先通过语义字号、字重和留白建立，不依赖自定义字体堆叠

### 文案原则
- 简洁明了，避免冗余
- 使用用户熟悉的语言，避免技术术语
- 操作按钮使用动词：保存、删除、取消
- 错误提示说明原因和解决方案

### 标点符号
- 中文文案使用全角标点：，。！？
- 英文/数字与中文之间加空格：使用 Qt 6.5 开发
- 省略号使用：…（而非 ...）
- 避免使用感叹号（除非强调）

### 占位符文本
- 使用"请输入..."格式
- 提供具体示例：请输入邮箱（example@domain.com）
- 避免使用"在此输入"等模糊表述

## 图标规范

### 图标尺寸
- 小图标：16px（列表项、按钮）
- 标准图标：24px（工具栏、菜单）
- 大图标：32px（标题、特性展示）
- 超大图标：48px+（空状态、引导页）

### 图标使用
- 16px 的功能图标（如按钮、菜单项、列表项文字前的小图标）默认从网上下载 SVG 资源，不再以系统图标主题或 DTK Icons 作为优先来源
- 下载的 16px 功能图标必须在视觉风格上保持与目标系统风格一致或接近一致，避免混入明显不同语言的图标集合
- 网络图标必须使用 SVG 格式，并保持单色黑白语义，不使用依赖源文件固有配色的多色图标
- SVG 功能图标必须通过 alpha 着色方式上色，使用时应绑定与同级文字一致的语义颜色，确保图标与文字颜色保持一致
- 不要把 16px 功能图标先转成 PNG 等位图资源再着色；这会破坏图标与文字同色的约束，并导致选中、悬停、禁用态颜色不可靠
- 保持图标风格一致（线性/填充）
- 图标颜色跟随文本颜色
- 交互图标的默认 tint 应跟随 `Theme.iconNormal`，而不是 `Theme.textMuted`
- 导航、按钮、菜单、列表项中的选中图标应与同级选中文字共用同一语义前景变量，而不是只做“接近”的色彩处理
- 重要操作使用彩色图标（删除用红色）

### 图标命名
- 使用 kebab-case：`arrow-right`, `file-text`, `user-circle`
- 方向性图标：`chevron-left/right/up/down`
- 状态图标：`check-circle`, `alert-triangle`, `info-circle`

## 数据展示

### 空状态
- 必须提供空状态提示
- 包含图标 + 说明文字 + 操作按钮（可选）
- 说明原因：暂无数据、搜索无结果、权限不足

### 加载状态
- 短时加载（<1s）：使用 BusyIndicator
- 长时加载（>1s）：使用 ProgressBar + 百分比
- 列表加载：使用 Skeleton 骨架屏
- 避免阻塞整个界面

### 错误处理
- 表单错误：字段下方显示红色错误文本
- 操作错误：使用 Toast 提示
- 严重错误：使用 DTK Dialog 对话框
- 网络错误：提供重试按钮

## 表单规范

### 表单布局
- 标签在输入框上方（垂直布局）
- 必填字段标记红色星号 *
- 相关字段分组，组间距 24px
- 表单最大宽度 600px

### 输入验证
- 实时验证：失焦时验证
- 即时反馈：显示错误/成功状态
- 提交验证：阻止无效表单提交
- 错误聚焦：自动滚动到第一个错误字段

### 按钮位置
- 主按钮在右侧（保存、提交）
- 次要按钮在左侧（取消、重置）
- 危险操作使用二次确认

## 状态管理

### 禁用状态
- 降低不透明度至 0.4
- 禁用鼠标交互
- 提供禁用原因（Tooltip）

### 选中状态
- 使用主题色背景或边框
- 提供视觉反馈（勾选图标）
- 支持多选时显示选中数量

### 激活状态
- 标签页/导航项使用下划线或背景色
- 保持激活状态持久化
- 刷新页面后恢复激活状态

## 响应式设计

### 断点策略
- 小屏（<768px）：单栏布局，隐藏侧边栏
- 中屏（768-1200px）：双栏布局，可折叠侧边栏
- 大屏（>1200px）：三栏布局，固定侧边栏

### 移动端适配
- 触摸目标最小 44x44px
- 增大间距至 16px
- 使用抽屉替代侧边栏
- 简化导航层级

## 国际化

### 文本处理
- 所有文本使用 `qsTr()` 包裹
- 避免硬编码文本
- 预留 30% 文本扩展空间（翻译）

### 布局适配
- 使用 `Layout.fillWidth` 而非固定宽度
- 避免假设文本长度
- RTL 语言支持（阿拉伯语、希伯来语）

## 代码规范

### 属性顺序
1. id
2. 继承属性（width, height, color）
3. 自定义属性（property）
4. 信号（signal）
5. 子元素
6. 状态和过渡（states, transitions）
7. 函数（function）

### 性能优化
- 避免深层嵌套（最多 5 层）
- 使用 `Loader` 延迟加载复杂组件
- 绑定表达式保持简单，复杂逻辑使用函数
- 避免在循环中创建对象

### 注释规范
- 复杂逻辑必须添加注释
- 使用 `//` 单行注释
- 组件顶部添加功能说明
- 标记 TODO/FIXME/HACK
