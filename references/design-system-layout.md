# 设计系统 - 窗口布局规范

> 标准主窗口默认优先保留窗管装饰和标准窗口行为；如果目标平台支持统一头部视觉且不破坏这些窗管能力，则采用标题栏与工具栏一体化布局，否则保留系统标题栏并将工具栏紧贴其下方。

## 应用窗口结构

### 标准应用布局（带侧边栏）

#### Logo
- 应用主窗口左上角保留应用 Logo 或主入口区域。
- 常见视觉尺寸可参考 `32px`，并与窗口左边距、标题栏高度一起形成稳定的起始节奏。
- 如果 Logo 同时承担首页返回或主菜单入口，需要让点击语义和拖动区域明确分离。

#### 窗口结构
- 带左侧边栏的桌面应用通常采用左右结构。
- 右侧上方是标题栏与工具栏一体区，下方是主内容区，底部可选状态栏。
- 右上角保留菜单、最小化、最大化 / 还原、关闭等窗口控制按钮。
- 侧边栏默认允许展开与收起，并在紧凑窗口中优先收缩侧边栏，而不是先压缩主内容可读性。

```
┌────────┬────────────────────────────────────┐
│          │  标题栏 (40px)               ☰  ─  □  ✕  │
│  [Logo]  ├────────────────────────────────────┤
│  (32px)  │                                          │
│          │                                          │
│  侧边栏   │           主内容区域                       │
│ (240px)  │                                          │
│ [毛玻璃]  │                                          │
│          │                                          │
│          │                                          │
└─────────┴───────────────────────────────────┘
```

## 常见布局模式

### 1. 单栏布局（Simple）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├─────────────────────────────────────────────────┤
│                                                 │
│          ┌─────────────────────┐                │
│          │                     │                │
│          │    内容区域         │                │
│          │   (max 600px)       │                │
│          │                     │                │
│          └─────────────────────┘                │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 适用于设置页面、引导流程、简单表单和需要用户聚焦单一任务的界面。
- 内容区通常居中布置，并限制最大宽度，避免在宽屏上形成过长文本行或过空的表单布局。
- 页面层级主要依靠标题、分组留白和按钮区完成，不需要额外的导航列。
- 当内容列、小卡片组或表单列没有占满整块内容区时，应把它们作为一个整体水平居中，而不是默认贴左或贴右。

### 2. 侧边栏布局（Sidebar）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  侧边栏  │                                      │
│ (240px)  │         主内容区域                    │
│  [毛玻璃] │                                      │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

- 适用于文件管理器、笔记应用、媒体管理、设置中心等需要稳定一级导航的桌面应用。
- 侧边栏承担分区切换、筛选入口和少量状态信息；主内容区承担列表、详情或编辑流程。
- 当侧边栏承担桌面应用的持久主导航时，这个布局不是参考图，而是强制结构基线：左侧导航列、右侧主内容区，默认启动即进入该分栏布局。
- 当侧边栏承担桌面应用的持久主导航时，侧边栏表面必须使用已验证的窗管或 compositor blur 路径。纯色背景只能作为兼容回退，不能把 blur 当成可选项。
- 右侧主内容区中的可见内容应整体相对该内容区居中；如果设置了最大宽度或固定列宽，左右外边距必须保持一致。
- 如果主内容区首屏由多个卡片共同组成一个横向概览区，必须先定义列数和跨度，再摆放卡片；不要把大卡片和小卡片随意堆砌成边界不齐的松散布局。
- 当一列使用单张大卡片、另一列使用多张堆叠卡片时，堆叠列的总高度应与大卡片对齐，并保持两列的外边界在顶部和底部都闭合。
- 横向概览区中的卡片宽度和高度可以不同，但必须服务于明确的网格关系，而不是让视觉只在某一条边上对齐。

### 横向概览与仪表盘组合

```
┌──────────────────────┬──────────────────────┐
│                      │  指标卡 1            │
│      大卡片          ├──────────────────────┤
│    (2x2 / 3x2)       │  指标卡 2            │
│                      ├──────────────────────┤
│                      │  指标卡 3            │
└──────────────────────┴──────────────────────┘
```

- 适用于健康评分、系统概览、资源监控和首屏摘要等需要“主指标 + 辅助指标”同时展示的界面。
- 大卡片与右侧堆叠卡片要共享同一个外轮廓盒子：顶部齐、底部齐、左右边界齐。
- 若右侧卡片数量或内容高度变化，优先回到网格重新分配行高、跨度和最小高度，而不是接受某一侧明显更高或更矮。
- 同一组合中的卡片间距应保持一致，不要在同一横向带里混用多种间距。

### 横向标题区与视觉重心

```
┌──────────────────────────────────────────────┐
│ 标题 + 说明        主操作        状态/摘要    │
└──────────────────────────────────────────────┘
```

- 适用于首页概览、列表页顶部、分析页头部和带主操作按钮的摘要区域。
- 标题、说明、主按钮、状态摘要应共同构成横向平衡，而不是把视觉重量全部压在左侧或中间后让另一端留空。
- 如果左侧已有大标题和时间说明，且中间或左侧存在主按钮，则右侧应放置与之相称的状态块、筛选块、次级操作或统计摘要，避免形成明显的“头重脚轻”或“一边重一边空”。
- 若业务上确实没有右侧内容，应收缩该横向带的可用宽度或改变主按钮位置，而不是保留一个仅起到制造空白的大块占位区。

### 3. 三栏布局（Three Column）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├────────┬──────────┬───────────────────────────────┤
│        │          │                              │
│ 左侧栏 │  中间栏  │                              │
│(200px) │ (280px)  │        右侧内容区             │
│ [导航] │  [列表]  │                              │
│        │          │                              │
└────────┴──────────┴───────────────────────────────┘
```

- 适用于邮件客户端、知识管理工具、文档管理和需要“导航 + 列表 + 详情”三级结构的产品。
- 左栏负责大类导航，中栏负责二级列表或结果集合，右栏负责内容详情或编辑区域。
- 当窗口宽度不足时，应优先折叠左栏或把中栏切换为覆盖式面板，而不是让三列同时挤压。

### 4. 主从布局（Master-Detail）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├──────────────┬──────────────────────────────────┤
│              │                                  │
│   主列表     │                                  │
│   (30%)      │         详情区域 (70%)           │
│              │                                  │
│              │                                  │
└──────────────┴──────────────────────────────────┘
```

- 适用于设置页面、属性编辑器、配置列表和详情页。
- 左侧主列表负责项目切换，右侧详情区负责参数展示、说明或编辑。
- 当数据密度较高时，主列表建议保持固定宽度范围，详情区使用弹性宽度。

### 5. 浏览器布局（Browser）

```
┌─────────────────────────────────────────────────┐
│  标签栏 (38px)    [标签1] [标签2] [+]           │
├─────────────────────────────────────────────────┤
│  导航栏 (44px)    ← → ⟳  [地址栏]      ⋮       │
├─────────────────────────────────────────────────┤
│  书签栏 (32px)    [书签1] [书签2] [书签3]       │
├─────────────────────────────────────────────────┤
│                                                 │
│              网页内容区域                        │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 适用于浏览器、多标签文档应用和需要多层导航的桌面工具。
- 顶部结构通常由标签栏、导航栏和内容区纵向分层组成，必要时再追加书签栏、信息栏或状态栏。
- 标签、地址输入、全局操作和窗口控制按钮要在顶部形成清晰层级，避免多个工具条同时争夺主视觉位置。
