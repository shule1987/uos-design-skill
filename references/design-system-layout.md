# 设计系统 - 窗口布局规范

> 标准主窗口默认采用标题栏与工具栏二合一的自绘标题栏，以保持统一的应用风格；仅在平台限制明显或窗口类型不适合自绘时退回系统装饰。

## 应用窗口结构

### 标准应用布局（带侧边栏）

#### logo
- 应用主窗口必须有应用logo，在应用窗口左上角
- 大小32px
- 距离窗口左边10px
#### 窗口结构
- 带左侧边栏的，应用布局为左右结构
- 右侧上方为标题栏与工具栏一体区，下方是主要的内容区，最底下为状态栏（可选）
- 右上角必须要菜单、最小化、最大化（可选）、关闭按钮
- 侧边栏可以展开收起

```
┌────────┬────────────────────────────────────┐
│          │  标题栏 (40px)               ☰  ─  □  ✕  │
│  [Logo]  ├────────────────────────────────────┤
│  (32px)  │                                          │
│          │                                          │
│  侧边栏   │           主内容区域                       │
│ (240px)  │                                          │
│ [毛玻璃]  │                                          │
│          │                                          │
│          │                                          │
└─────────┴───────────────────────────────────┘
```

## 常见布局模式

### 1. 单栏布局（Simple）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├─────────────────────────────────────────────────┤
│                                                 │
│          ┌─────────────────────┐                │
│          │                     │                │
│          │    内容区域         │                │
│          │   (max 600px)       │                │
│          │                     │                │
│          └─────────────────────┘                │
│                                                 │
└─────────────────────────────────────────────────┘
```

> 示例描述：这里给出“1. 单栏布局（Simple）”的实现思路，适用：设置页面、简单表单。整体以 `Item` 作为根节点，内部主要组织 `Column` 等内容。

### 2. 侧边栏布局（Sidebar）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  侧边栏  │                                      │
│ (240px)  │         主内容区域                    │
│  [毛玻璃] │                                      │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

> 示例描述：这里给出“2. 侧边栏布局（Sidebar）”的实现思路，适用：文件管理器、笔记应用。整体以 `Row` 作为根节点，内部主要组织 `Rectangle` 和 `Item` 等内容。 关键尺寸上间距为 `0`；行为上覆盖可选模糊效果。

### 3. 三栏布局（Three Column）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├────────┬──────────┬───────────────────────────────┤
│        │          │                              │
│ 左侧栏 │  中间栏  │                              │
│(200px) │ (280px)  │        右侧内容区             │
│ [导航] │  [列表]  │                              │
│        │          │                              │
└────────┴──────────┴───────────────────────────────┘
```

> 示例描述：这里给出“3. 三栏布局（Three Column）”的实现思路，适用：邮件客户端、笔记应用。整体以 `Row` 作为根节点，内部主要组织 `Rectangle` 和 `Item` 等内容。 关键尺寸上间距为 `0`。

### 4. 主从布局（Master-Detail）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├──────────────┬──────────────────────────────────┤
│              │                                  │
│   主列表     │                                  │
│   (30%)      │         详情区域 (70%)           │
│              │                                  │
│              │                                  │
└──────────────┴──────────────────────────────────┘
```

> 示例描述：这里给出“4. 主从布局（Master-Detail）”的实现思路，适用：设置页面、详情页。整体以 `Row` 作为根节点，内部主要组织 `ListView`、`Rectangle` 和 `Item` 等内容。 关键尺寸上间距为 `0`；行为上覆盖列表承载。

### 5. 浏览器布局（Browser）

```
┌─────────────────────────────────────────────────┐
│  标签栏 (38px)    [标签1] [标签2] [+]           │
├─────────────────────────────────────────────────┤
│  导航栏 (44px)    ← → ⟳  [地址栏]      ⋮       │
├─────────────────────────────────────────────────┤
│  书签栏 (32px)    [书签1] [书签2] [书签3]       │
├─────────────────────────────────────────────────┤
│                                                 │
│              网页内容区域                        │
│                                                 │
└─────────────────────────────────────────────────┘
```

> 示例描述：这里给出“5. 浏览器布局（Browser）”的实现思路，适用：浏览器、多标签文档应用和需要多层导航的桌面工具。整体采用标签栏、导航栏和内容区的纵向分层结构，必要时可追加书签栏或状态栏。
