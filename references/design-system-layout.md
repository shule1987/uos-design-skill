# 设计系统 - 窗口布局规范

> 标准主窗口默认采用标题栏与工具栏二合一的自绘标题栏，以保持统一的应用风格；仅在平台限制明显或窗口类型不适合自绘时退回系统装饰。

## 应用窗口结构

### 标准应用布局（带侧边栏）

#### Logo
- 应用主窗口左上角保留应用 Logo 或主入口区域。
- 常见视觉尺寸可参考 `32px`，并与窗口左边距、标题栏高度一起形成稳定的起始节奏。
- 如果 Logo 同时承担首页返回或主菜单入口，需要让点击语义和拖动区域明确分离。

#### 窗口结构
- 带左侧边栏的桌面应用通常采用左右结构。
- 右侧上方是标题栏与工具栏一体区，下方是主内容区，底部可选状态栏。
- 右上角保留菜单、最小化、最大化 / 还原、关闭等窗口控制按钮。
- 侧边栏默认允许展开与收起，并在紧凑窗口中优先收缩侧边栏，而不是先压缩主内容可读性。

```
┌────────┬────────────────────────────────────┐
│          │  标题栏 (40px)               ☰  ─  □  ✕  │
│  [Logo]  ├────────────────────────────────────┤
│  (32px)  │                                          │
│          │                                          │
│  侧边栏   │           主内容区域                       │
│ (240px)  │                                          │
│ [毛玻璃]  │                                          │
│          │                                          │
│          │                                          │
└─────────┴───────────────────────────────────┘
```

## 常见布局模式

### 1. 单栏布局（Simple）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├─────────────────────────────────────────────────┤
│                                                 │
│          ┌─────────────────────┐                │
│          │                     │                │
│          │    内容区域         │                │
│          │   (max 600px)       │                │
│          │                     │                │
│          └─────────────────────┘                │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 适用于设置页面、引导流程、简单表单和需要用户聚焦单一任务的界面。
- 内容区通常居中布置，并限制最大宽度，避免在宽屏上形成过长文本行或过空的表单布局。
- 页面层级主要依靠标题、分组留白和按钮区完成，不需要额外的导航列。

### 2. 侧边栏布局（Sidebar）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  侧边栏  │                                      │
│ (240px)  │         主内容区域                    │
│  [毛玻璃] │                                      │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

- 适用于文件管理器、笔记应用、媒体管理、设置中心等需要稳定一级导航的桌面应用。
- 侧边栏承担分区切换、筛选入口和少量状态信息；主内容区承担列表、详情或编辑流程。
- 侧边栏可以使用纯色面板，也可以在性能允许时增加轻度 blur，但 blur 不是必选项。

### 3. 三栏布局（Three Column）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├────────┬──────────┬───────────────────────────────┤
│        │          │                              │
│ 左侧栏 │  中间栏  │                              │
│(200px) │ (280px)  │        右侧内容区             │
│ [导航] │  [列表]  │                              │
│        │          │                              │
└────────┴──────────┴───────────────────────────────┘
```

- 适用于邮件客户端、知识管理工具、文档管理和需要“导航 + 列表 + 详情”三级结构的产品。
- 左栏负责大类导航，中栏负责二级列表或结果集合，右栏负责内容详情或编辑区域。
- 当窗口宽度不足时，应优先折叠左栏或把中栏切换为覆盖式面板，而不是让三列同时挤压。

### 4. 主从布局（Master-Detail）

```
┌─────────────────────────────────────────────────┐
│  标题栏                                          │
├──────────────┬──────────────────────────────────┤
│              │                                  │
│   主列表     │                                  │
│   (30%)      │         详情区域 (70%)           │
│              │                                  │
│              │                                  │
└──────────────┴──────────────────────────────────┘
```

- 适用于设置页面、属性编辑器、配置列表和详情页。
- 左侧主列表负责项目切换，右侧详情区负责参数展示、说明或编辑。
- 当数据密度较高时，主列表建议保持固定宽度范围，详情区使用弹性宽度。

### 5. 浏览器布局（Browser）

```
┌─────────────────────────────────────────────────┐
│  标签栏 (38px)    [标签1] [标签2] [+]           │
├─────────────────────────────────────────────────┤
│  导航栏 (44px)    ← → ⟳  [地址栏]      ⋮       │
├─────────────────────────────────────────────────┤
│  书签栏 (32px)    [书签1] [书签2] [书签3]       │
├─────────────────────────────────────────────────┤
│                                                 │
│              网页内容区域                        │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 适用于浏览器、多标签文档应用和需要多层导航的桌面工具。
- 顶部结构通常由标签栏、导航栏和内容区纵向分层组成，必要时再追加书签栏、信息栏或状态栏。
- 标签、地址输入、全局操作和窗口控制按钮要在顶部形成清晰层级，避免多个工具条同时争夺主视觉位置。
