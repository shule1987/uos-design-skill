# Waiver Policy

Load this file when an exception is necessary.

- Use narrow waiver comments in touched files when an exception is necessary.
- Format waiver comments as `uos-design: allow-...` and include the exact platform, version, or component reason.
- Reuse established waiver names when possible. Common examples:
  - `uos-design: allow-frameless`
  - `uos-design: allow-popup-window`
  - `uos-design: allow-custom-window-buttons`
  - `uos-design: allow-offset-window-button-group`
  - `uos-design: allow-customized-window-flags`
  - `uos-design: allow-dtk-template-override`
  - `uos-design: allow-transparent-main-window`
  - `uos-design: allow-full-window-base-under-sidebar-blur`
  - `uos-design: allow-full-window-blur`
