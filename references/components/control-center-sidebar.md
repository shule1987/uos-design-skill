# 控制中心式左侧边栏基线规则

用于所有带持久主导航左侧栏的桌面应用。该文档以 `dde-control-center` 的当前实现为来源，但在 `uos-design` 中应被当作强制布局基线，而不是仅在用户点名控制中心时才参考的可选先例。

## 来源
- 主窗口结构：`src/dde-control-center/plugin/DccWindow.qml`
- 二级页侧栏：`src/dde-control-center/plugin/SecondPage.qml`
- 主页网格：`src/dde-control-center/plugin/HomePage.qml`
- 分组列表：`src/dde-control-center/plugin/DccGroupView.qml`
- 菜单项：`src/dde-control-center/plugin/DccMenuItem.qml`
- 导航模型：`src/dde-control-center/navigationmodel.cpp`
- 侧栏默认宽度持久化：`src/dde-control-center/dccmanager.cpp`
- 组件说明：`docs/v25-dcc-interface.zh_CN.md`

## 观察到的实现规律

### 1. 源码里有 HomePage，但默认运行逻辑是“常驻左侧栏”
- 控制中心主窗口结构确实包含两个视图：
  - `HomePage.qml`：网格入口页
  - `SecondPage.qml`：左侧导航 + 右侧内容的分栏布局
- 但默认运行逻辑在 `dccmanager.cpp` 里已经写明 `禁用首页`：
  - `tryShow()` 在首次展示时调用 `showPage(m_root, QString())`
  - `doShowPage()` 里如果目标是 `m_root`，会直接改成 `m_root->getChildren().first()`
- 这意味着控制中心的实际启动体验应理解为：
  - 默认直接进入首个模块
  - 默认看到常驻左侧栏
  - `HomePage.qml` 不能当成当前产品默认入口先例
- 因此，对所有带持久主导航左侧栏的桌面应用，默认应理解为“启动即常驻左侧栏的设置分栏布局”，而不是先做一个网格首页再进入二级页。

### 2. 顶层窗口保留 DTK/窗管语义
- 主窗体使用 `D.ApplicationWindow` 和 `D.TitleBar`。
- `flags` 仍保留标准窗口按钮与标题栏语义，没有走默认无边框路径。
- 侧栏表面通过 `D.StyledBehindWindowBlur` 或等效系统路径形成已验证的 blur/tint 结果，右侧内容区则使用明确的主题底面；不是靠一整块全窗 blur，或给侧栏手搓一层假毛玻璃。

### 3. 二级页布局是“左栏 + 1px 分隔 + 右侧内容”
- `SecondPage.qml` 的结构是：
  - 左侧 `Item`：顶部搜索框 + 导航 `ListView`
  - 中间 `Rectangle`：`1px` splitter
  - 右侧 `Rectangle` + `StackView`：内容区域
- 这属于很典型的控制中心式设置分栏布局，比“卡片漂浮在大画布上”更偏工具型桌面设置应用。

### 4. 侧栏宽度可拖拽、可隐藏、会记忆
- 侧栏宽度通过 `DccApp.sidebarWidth` 持久化。
- 默认宽度不是一个绝对常量：
  - 英文环境默认 `210`
  - 其他语言默认 `180`
- 拖拽时如果宽度小于 `110`，侧栏直接隐藏到 `0`，不是变成固定 `60px` 图标轨。
- 标题栏左侧有一个 sidebar 切换按钮，可在“隐藏”和“恢复上次宽度”之间切换。
- 右侧内容区保持最小宽度约 `510-520px`，侧栏调整不能压垮主内容区。
- 这个切换入口在控制中心里读作一个独立的方形 sidebar-toggle 按钮，使用专用的侧栏展开/收起图标，不是一个裸露的 `chevron-left`、`chevron-right`、返回箭头或文本按钮。
- 左上角应用 logo 读作一个稳定的窗体锚点。侧栏展开和收起时，logo 相对窗口左上角的位置不应跳变，不应从左栏里消失再在右侧顶带里重现，也不应跟着侧栏宽度动画一起横向漂移。

### 5. 侧栏内容有稳定的 10px 外侧内边距
- `SecondPage.qml` 里的 `ListView` 使用：
  - `leftMargin: 10`
  - `rightMargin: 10`
  - `topMargin: 6`
  - `bottomMargin: 10`
- 搜索框也使用 `10px` 左右边距。
- 这和 `uos-design` 里“持久侧栏的导航列表距侧栏背景左右各 10px”是对齐的。

### 6. 侧栏项本身比通用文档更紧凑
- 控制中心二级页的列表 delegate 在 `SecondPage.qml` 里使用 `D.ItemDelegate`，`implicitHeight: 40`。
- `DccMenuItem.qml` 的内容内边距更具体：
  - `leftPadding: 14`
  - `rightPadding: 10`
  - 默认右侧箭头 affordance
- 图标常用 `24px`，描述文字存在时由 `DccEditorItem` 根据内容提高行高。

### 7. 键盘导航不是补丁，而是页面容器职责
- `SecondPage.qml` 的 `ListView` 直接处理 `Up`、`Down`、`Enter/Return`。
- `DccGroupView.qml` 和 `DccEditorItem.qml` 也显式处理焦点链和上下导航。
- 如果要复刻控制中心式侧栏，键盘导航、焦点回流、鼠标点击后夺回列表焦点都应作为默认能力，而不是后补。

### 8. 左侧栏 blur 的强制规则不是“自绘一块假毛玻璃”
- 对这类应用，必须满足的规则是：
  - 侧栏使用已验证的 `StyledBehindWindowBlur` 或等效系统 blur 路径
  - 右侧内容区保持明确的主题底面
  - 两侧再通过结构、分隔线和轻度背景差异区分
- 不要为了修复侧栏和右侧内容区的缝隙、拖拽区或分隔线问题，在整窗或整个分栏壳子下面再铺一层 `Theme.bg`、`Theme.panelBg` 或其他不透明主题底面。那会直接把 blur 的可见结果压成实色面板。
- 不要把这条规则误读成“左栏必须单独叠一块自绘半透明 Rectangle”。真正需要的是已验证的窗管、compositor 或 DTK blur 路径，而不是手搓一层假 blur。

### 9. 视觉结果必须接近控制中心，而不是“只要代码里写了 blur 就算完成”
- 浅色主题下，侧栏应读作接近白色玻璃面板，不应呈现明显浅蓝、青白、灰蓝卡片感。
- 深色主题下，侧栏应读作接近中性近黑玻璃面板，不应呈现海军蓝、墨蓝、青黑或其他带明显色偏的深色板块。
- 如果运行结果里几乎看不出 blur，只剩一块高透明度或高不透明度的实色背景，也应判定为未完成。
- 如果为了解决中缝、拖拽条或底面问题而在 blur 侧栏背后加了一整块全窗背景，导致 blur 只剩带 tint 的纯色板，也应判定为未完成。
- 这类问题不能用“代码里已经有 `StyledBehindWindowBlur`”来辩解。以实际运行观感为准。

### 10. 不可再自由发挥的实现边界
- 不要把侧栏表面改成任意自创配色。默认必须回到 `references/foundations/colors.md` 里的中性浅 / 深基线。
- 不要把控制中心式侧栏理解成“任意左栏 + 任意右栏”。它的关键不是只有结构，还包括工具型桌面设置应用的视觉克制：
- 侧栏是中性 blur 面板
- 中间是细分隔线
- 右侧是明确内容底面
- 不靠强烈品牌色或大面积彩色背景建立结构
- 右侧底面只属于右侧内容区，不得覆盖到 blur 侧栏背后
- 不要因为产品页面里用了绿色、橙色、红色状态色，就把侧栏、主背景或标题带整体染色。
- 控制中心式的合一顶带本身不承担页面标题展示职责。顶带里保留 logo、侧栏切换、主菜单和窗管按钮；模块或页面标题应放在页面内容头部，除非产品明确要求顶带展示标题。
- 合一顶带的控件视觉必须沿用 DTK 标准样式，不要自己画一套标题带按钮、胶囊、边框或菜单触发器外观。
- 如果本机就有控制中心可运行，应在同一主题模式下把实现结果与控制中心并排比对；看起来不像控制中心的侧栏基线，就说明 skill 没落地。

## 强制继承项
- 启动即进入常驻左侧栏的分栏布局。
- 采用左栏 + `1px` 分隔 + 右侧内容区的左右结构，而不是顶部主导航或上下分栏。
- 左侧栏使用已验证的 blur 路径，而不是纯色替代或手工伪造 blur。
- 左侧栏的视觉结果必须接近控制中心的中性玻璃侧栏，而不是带明显色偏的实色卡片。
- 侧栏导航列表与项距侧栏背景左右各 `10px`。
- 侧栏宽度可调并可记忆；宽度过小时直接隐藏，而不是退化为固定 `60px` 图标轨。
- 标题栏提供显式的侧栏展开/收起入口，并沿用控制中心式的独立方形 sidebar-toggle 按钮和专用 toggle 图标；不要退化成方向箭头、返回箭头或普通文字按钮。
- 左上角 logo 在侧栏展开/收起过程中保持窗口相对坐标稳定，不得通过双实例切换或位置跳变来“补”状态。
- 键盘上下选择、回车进入和焦点回流是内建行为，不是后补功能。

## 非强制继承项
- 默认 `180/210px` 宽度。
- 二级页必须带搜索框。
- 首页必须是控制中心那种网格入口。
- 列表项必须固定 `40px` 高度。
- 图标必须固定 `24px`。
