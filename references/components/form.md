# 表单组件

## Form

```
┌─────────────────────────────────────┐
│  用户名 *                            │
│  ┌───────────────────────────────┐  │
│  │ 请输入用户名                   │  │
│  └───────────────────────────────┘  │
│                                     │
│  邮箱 *                              │
│  ┌───────────────────────────────┐  │
│  │ 请输入邮箱                     │  │
│  └───────────────────────────────┘  │
│  ⚠ 邮箱格式不正确                   │
│                                     │
│  密码 *                              │
│  ┌───────────────────────────────┐  │
│  │ ••••••••                       │  │
│  └───────────────────────────────┘  │
│                                     │
│  ┌──────┐  ┌──────┐                │
│  │ 提交 │  │ 取消 │                │
│  └──────┘  └──────┘                │
└─────────────────────────────────────┘
```

- 用途：用于组织一组相关输入控件、说明文字和操作按钮。
- 布局：优先使用纵向分组和稳定的区块间距；桌面宽屏下也不要把标签和输入区拉得过散。
- 规则：长表单应分组、分段或分步，而不是简单堆成长页面。

## FormItem
- 用途：作为表单中的单个字段单元，承载标签、输入控件、说明和错误信息。
- 结构：通常由标签区、内容区和状态说明区组成；必填标记和错误提示要与标签形成稳定关系。
- 交互：错误信息优先贴近字段显示，避免只在表单底部统一报错。

## 使用建议
- 常规表单宽度应根据阅读与输入效率决定，不必无限拉宽。
- 提交按钮与取消按钮通常放在表单底部，并保持主次关系稳定。
