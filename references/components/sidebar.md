# 侧边栏组件

## Sidebar
- 用途：用于一级导航、资源分组、筛选入口和少量全局操作。
- 当侧边栏承担桌面应用的主导航时，必须采用控制中心式左右结构：左侧为侧边栏导航列，右侧为主内容区。
- 结构：通常由顶部品牌区或标题区、导航项列表、可选的折叠按钮组成；如果存在折叠按钮，其位置应在 logo 右侧 `20px` 处。
- 尺寸与状态：展开宽度通常在 `200-240px`，并可按内容和窗口宽度调节；最小可调宽度为 `100px`。不要把 `60px` 图标栏当作默认或唯一折叠目标。
- 视觉：当侧边栏承担桌面应用的主导航时，侧边栏表面必须使用目标环境中已验证的窗管或 compositor 模糊路径。纯色回退只能作为兼容回退，不能把 blur 当成可选样式。不要用自绘半透明层、截图模糊或本地 shader 伪造该 blur。
- 交互：支持 hover、选中、键盘导航和无障碍名称；当导航条目较多时建议支持折叠，但不是所有侧边栏都必须强制提供。若支持折叠，则在宽度缩到 `100px` 或主动触发折叠按钮时应隐藏侧边栏，并以短时动画完成展开与收起。

## 控制中心式主导航基线
- 任何带持久主导航左侧栏的桌面应用，都必须额外读取 `control-center-sidebar.md`。
- `control-center-sidebar.md` 不是可选特例，而是这类应用的强制结构基线。
- 默认启动行为应理解为“启动即进入常驻左侧导航分栏布局”，不要因为源码里仍有 `HomePage.qml` 就默认先做网格首页。
